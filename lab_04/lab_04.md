# 2. Архитектура системы

## Схема архитектуры

Клиент → Координатор → Сервер 1 / Сервер 2
<img width="1475" height="789" alt="image" src="https://github.com/user-attachments/assets/3360bef8-81aa-4628-a39c-e10928023126" />

## Компоненты системы

### 1. Клиент (client.py)

**Назначение:**

* шифрование пользовательских сообщений с помощью Fernet;
* отправка данных координатору;
* получение отсортированного списка сообщений;
* выбор типа сортировки (asc или desc).

**Порт:** отсутствует (инициирует соединение)

**Протокол:** HTTP → координатор (порт 8000)

---

### 2. Координатор (coordinator.py)

**Назначение:**

* принимает запросы клиента;
* перенаправляет запросы на доступный сервер;
* обеспечивает отказоустойчивость системы;
* выполняет переключение на резервный сервер при отказе основного.

**Порт:** 8000

**Протокол:** HTTP (клиент ↔ координатор), HTTPS + mTLS (координатор ↔ сервер)

---

### 3. Сервер 1 (server.py)

**Назначение:**

* прием зашифрованных сообщений;
* расшифровка данных;
* хранение сообщений;
* выполнение сортировки данных.

**Порт:** 5001

**Протокол:** HTTPS + mTLS

**Роль:** основной сервер

---

### 4. Сервер 2 (server.py)

**Назначение:**

* резервная обработка запросов;
* хранение сообщений;
* выполнение сортировки данных.

**Порт:** 5002

**Протокол:** HTTPS + mTLS

**Роль:** резервный сервер

---

### 5. Центр сертификации (CA)

**Назначение:**

* создание и подпись сертификатов сервера и клиента.

**Файлы:**

* ca_cert.pem
* ca_key.pem

---

### 6. Ключ Fernet

**Назначение:**

* симметричное шифрование сообщений.

**Файл:**

* encryption_key.txt

**Алгоритм:**

* Fernet (AES-128-CBC + HMAC-SHA256)

---

# 3. Инструкция по запуску

## Установка зависимостей

```bash
pip install flask cryptography requests
```

## Генерация сертификатов

```bash
chmod +x generate_certificates.sh
./generate_certificates.sh
```

Будут созданы файлы:

```text
ca_cert.pem
ca_key.pem
server_cert.pem
server_key.pem
client_cert.pem
client_key.pem
```

## Генерация ключа Fernet

```bash
python3 generate_key.py
```

Создается файл:

```text
encryption_key.txt
```

## Сохранение зависимостей

```bash
pip freeze > requirements.txt
```

## Запуск системы

### Терминал №1

```bash
python3 server.py 5001
```

### Терминал №2

```bash
python3 server.py 5002
```

### Терминал №3

```bash
python3 coordinator.py
```

### Терминал №4

```bash
python3 client.py
```

# 6. Выводы

В ходе выполнения лабораторной работы была разработана распределенная система с механизмами безопасности и отказоустойчивости.

В процессе работы были выполнены следующие задачи:

1. Настроена инфраструктура открытых ключей (PKI) и сгенерированы сертификаты X.509 для центра сертификации, серверов и клиента.

2. Реализована защищенная передача данных по протоколу HTTPS с использованием взаимной аутентификации (mTLS).

3. Для защиты передаваемой информации внедрено симметричное шифрование Fernet, обеспечивающее конфиденциальность и контроль целостности данных.

4. Реализованы основные компоненты системы:

   * клиент;
   * координатор;
   * два сервера обработки данных.

5. Настроен механизм отказоустойчивости. При недоступности основного сервера координатор автоматически перенаправляет запросы на резервный сервер без участия пользователя.

6. Выполнено индивидуальное задание варианта 14. Реализована сортировка данных при запросе к серверу. Пользователь может получать список сообщений в порядке возрастания (asc) и убывания (desc).

**Итог:** разработанная система обеспечивает защищенную передачу данных, взаимную аутентификацию участников, отказоустойчивость при сбоях серверов и поддержку сортировки данных в соответствии с требованиями варианта 14.
