# Задачи

## 1. Обработка данных IoT

Producer отправляет JSON с температурой.
gRPC сервис проверяет, находится ли температура в диапазоне 18–25°C, и возвращает:

* `"NORMAL"` — если температура в норме
* `"ALARM"` — если температура выходит за пределы нормы

---

## 2. Определение четности

Producer отправляет число.
gRPC сервис определяет, является ли число четным или нечетным, и возвращает:

* `"EVEN"` — четное число
* `"ODD"` — нечетное число

---

## 3. Слияние двух текстов

Producer отправляет два текста.
gRPC сервис объединяет тексты в одну строку и возвращает результат.

---

# Архитектура системы

```text
Producer → RabbitMQ → Consumer → gRPC Server
```

---

# Общая логика работы

1. Пользователь вводит сообщение в Producer
2. Producer отправляет сообщение в очередь RabbitMQ
3. Consumer получает сообщение из очереди
4. Consumer передает сообщение в gRPC сервис
5. gRPC сервер определяет тип задачи
6. Выполняется обработка данных
7. Результат возвращается Consumer
8. Consumer выводит результат пользователю
