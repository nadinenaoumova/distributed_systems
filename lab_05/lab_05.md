Цель работы

Научиться:

* запускать многоконтейнерные приложения;
* организовывать взаимодействие между сервисами;
* использовать Docker Compose для оркестрации;
* изменять бизнес-логику и инфраструктуру проекта;

Индивидуальное задание

Для варианта 14 необходимо:

* реализовать тему «Свободные места»;
* начать счетчик со значения 1000 и уменьшать его при каждом обращении к странице;
* изменить внешний порт с 8000 на 8888;
* использовать параметр `--no-cache-dir` при установке зависимостей в Dockerfile.

Бизнес-кейс

«Свободные места»

Необходимо создать веб-приложение:

* Flask отображает страницу;
* Redis хранит количество свободных мест;
* начальное значение составляет 1000;
* при каждом обновлении страницы количество свободных мест уменьшается на единицу.

Теоретические сведения

Docker Image

Docker Image — шаблон приложения, содержащий:

* исходный код;
* библиотеки;
* зависимости;
* настройки запуска.

Container

Контейнер — запущенный экземпляр Docker-образа.

Docker Compose

Docker Compose позволяет запускать несколько контейнеров одной командой через файл:

docker-compose.yml

Redis

Redis — быстрое in-memory хранилище данных.

В данной лабораторной работе Redis используется для хранения количества свободных мест веб-приложения.

Ход выполнения работы

1. Создание проекта

Была создана рабочая директория проекта:

mkdir lab_05

cd lab_05

2. Создание файла requirements.txt

Был создан файл requirements.txt со списком необходимых зависимостей:

Flask==2.0.1

Werkzeug==2.0.3

redis==4.6.0

Используемые библиотеки:

* Flask — создание веб-приложения;
* Werkzeug — библиотека для работы Flask;
* redis — подключение к Redis.

3. Создание файла app.py

Был создан файл app.py, содержащий код Flask-приложения.

Приложение выполняет следующие функции:

* принимает HTTP-запросы;
* подключается к Redis;
* хранит количество свободных мест;
* уменьшает количество свободных мест при каждом обращении;
* отображает текущее значение пользователю.

В рамках варианта 14 была реализована тема «Свободные места».

При первом запуске создается ключ:

places = 1000

После каждого обращения к странице выполняется команда:

cache.decr('places')

В результате количество свободных мест уменьшается:

1000 → 999 → 998 → 997 → ...

Пользователю отображается сообщение:

Свободных мест: X

4. Создание Dockerfile

Был создан файл Dockerfile для сборки Docker-образа Flask-приложения.

Содержимое файла:

FROM python:3.9-alpine

WORKDIR /code

COPY requirements.txt requirements.txt

RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["python", "app.py"]

В рамках индивидуального задания была изменена команда установки зависимостей:

RUN pip install --no-cache-dir -r requirements.txt

Параметр --no-cache-dir запрещает сохранение временного кэша pip, что уменьшает размер итогового Docker-образа.

5. Создание docker-compose.yml

Был создан файл docker-compose.yml.

Docker Compose используется для запуска нескольких контейнеров одной командой.

Содержимое файла:

services:

web:

```
build: .

ports:

  - "8888:5000"

depends_on:

  - redis
```

redis:

```
image: redis:alpine
```

В рамках варианта 14 был изменен внешний порт приложения:

8888:5000

6. Сборка и запуск проекта

Для сборки и запуска контейнеров использовалась команда:

docker compose up -d --build

После запуска Docker Compose:

* создает сеть проекта;
* запускает контейнер Redis;
* собирает Docker-образ Flask-приложения;
* запускает контейнер web.

Принцип работы приложения

После открытия страницы:

http://localhost:8888

происходит следующий процесс:

* браузер отправляет HTTP-запрос Flask-приложению;
* Flask получает запрос;
* Flask обращается к Redis;
* Redis проверяет наличие ключа places;
* при первом запуске создается значение 1000;
* Redis уменьшает значение на единицу;
* Flask получает новое значение;
* пользователю отображается количество свободных мест.

В результате Redis возвращал текущее количество свободных мест.
