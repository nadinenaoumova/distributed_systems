# 🔬 Методология

В работе использовалась симуляция Gossip-протокола для распределенной системы.

Система включает:

* 100 узлов;
* обмен gossip-сообщениями между узлами;
* случайные потери пакетов;
* моделирование отказов узлов;
* распространение информации о сбоях между узлами.

Исследовалось влияние следующих параметров:

* Gossip Interval — интервал между gossip-обменом;
* Gossip Fanout — количество соседей для рассылки;
* Packet Loss — процент потерянных сообщений;
* Node Failures — процент отказавших узлов.

Для моделирования использовались следующие симуляторы:

* SerfSimulator (Gossip);
* HeartbeatSimulator;
* PingSimulator.

# 📊 Теоретический расчет Bandwidth

Для оценки сетевой нагрузки использовалась функция расчета полосы пропускания.

## Формула расчета

Bandwidth = (N × Fanout × PacketSize) / Interval

где:

* N — количество узлов;
* Fanout — количество соседей;
* PacketSize — размер gossip-пакета;
* Interval — интервал gossip-обмена.

## Анализ графика

На графике представлено сравнение времени полной конвергенции системы при различных значениях Fanout.

### Наблюдения

* Fanout = 5 обеспечивает более быстрое распространение информации.
* При Fanout = 3 convergence происходит медленнее.
* Увеличение Fanout уменьшает время обнаружения отказов.

### Причина

Больший Fanout позволяет каждому узлу отправлять gossip-сообщения большему количеству соседей, поэтому информация о сбоях распространяется быстрее.
